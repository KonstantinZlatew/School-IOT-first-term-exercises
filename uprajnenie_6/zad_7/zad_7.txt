1. Избран микроконтролер

Отговор:
Избран микроконтролер: STM32F103

Обяснение:
STM32F103 е широко използван 32-битов микроконтролер, подходящ за обучение и IoT приложения. Той комбинира 
процесорно ядро, памети и богата периферия в един чип.

2. Основни компоненти на STM32F103
2.1 Процесорно ядро

Отговор:
Процесорно ядро: ARM Cortex-M3

Обяснение:
Cortex-M3 е 32-битово RISC ядро, оптимизирано за embedded системи. То изпълнява програмния код, обработва 
прекъсвания и управлява достъпа до паметта и перифериите. Подходящо е за реалновременни IoT приложения с ниска 
консумация.

2.2 Памет
Flash памет

Отговор:
Flash памет: 64 KB

Обяснение:
Flash паметта съхранява програмния код и константни данни. Тя е енергонезависима и съдържанието ѝ се запазва 
при изключване на захранването.

SRAM

Отговор:
SRAM памет: 20 KB

Обяснение:
SRAM се използва за променливи, стек и динамични данни по време на изпълнение на програмата. Тя е бърза, но 
енергозависима.

EEPROM

Отговор:
EEPROM: Няма хардуерна EEPROM (емулира се във Flash)

Обяснение:
STM32F103 няма отделна EEPROM. Данни, които трябва да се запазят, се съхраняват чрез софтуерна емулация във 
Flash паметта.

2.3 Периферни устройства
GPIO (General Purpose Input/Output)

Отговор:
GPIO пинове за вход и изход

Обяснение:
GPIO позволяват директно управление на външни устройства като LED, бутони и релета. В примера GPIO се използва 
за управление на LED (PC13) и четене на бутон (PA1).

ADC (Analog-to-Digital Converter)

Отговор:
12-битов ADC

Обяснение:
ADC преобразува аналогови сигнали в цифрови стойности. В кода ADC се използва за четене на аналогов сензор на 
пин PA0 и преобразуване в напрежение.

UART (Universal Asynchronous Receiver/Transmitter)

Отговор:
UART за серийна комуникация

Обяснение:
UART осигурява комуникация между микроконтролера и компютър или друг модул. В примера UART се използва за 
извеждане на диагностична информация в Serial Monitor.

I2C (Inter-Integrated Circuit)

Отговор:
I2C интерфейс (PB6 – SCL, PB7 – SDA)

Обяснение:
I2C позволява комуникация с множество външни сензори чрез два проводника. В примера се симулира сканиране за I2C 
устройства.

3. Схема на архитектурата на STM32F103

Отговор:
Блок-схема на архитектурата

Обяснение:

+--------------------------------------+
|           STM32F103 MCU               |
|                                      |
|  +-------------+                     |
|  | Cortex-M3   |                     |
|  | CPU Core    |                     |
|  +------+------+                     |
|         |                            |
|  +------+-----------+---------------+
|  |      Memory      |               |
|  |  Flash  | SRAM   |               |
|  +---------+--------+               |
|         |                            |
|  +------+---------------------------+
|  |          Peripherals             |
|  |  GPIO | ADC | UART | I2C | Timers|
|  +----------------------------------+


Процесорното ядро управлява паметта и периферните устройства чрез вътрешна шинна система. Всички компоненти 
са интегрирани в един чип.

4. Анализ на примерния код

Отговор:
Кодът демонстрира работа с основните периферии на STM32F103.

Обяснение:

GPIO: включване и изключване на LED

ADC: четене на аналогов сензор и изчисляване на напрежение

Digital Input: четене на бутон

UART: извеждане на данни към серийния монитор

I2C: симулация на комуникация с външни устройства

Това показва как различните хардуерни блокове на MCU работят заедно в реално приложение.

5. Заключение

Отговор:
STM32F103 представлява типичен микроконтролер с интегрирани ядро, памети и периферия, подходящ за IoT приложения.

Обяснение:
Чрез комбинирането на процесорно ядро, памет и комуникационни интерфейси в един чип, MCU като STM32F103 
позволяват изграждането на компактни, енергийно ефективни и надеждни IoT устройства. Разбирането на тази 
архитектура е ключово за правилния избор и използване на микроконтролери.
